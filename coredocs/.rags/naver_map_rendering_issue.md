# 네이버 지도 랜더링 및 연동 장애 분석 리포트

## 🕵️ 핵심 장애 원인

### 1. 런타임 크래시
지도가 하얗게 뜨며 앱 전체가 멈추는 현상이었습니다. 네이버 SDK 인증 실패 시 naver.maps 객체는 null이 되는데, 이 상태에서 LatLng 생성자를 호출하여 자바스크립트 실행이 중단되었습니다. 이 에러가 상담 기능까지 영향을 주어 대화가 마비되었습니다.

### 2. URI 민감도 및 포트 점유
NCP 설정 후에도 인증 실패가 반복되었습니다. 네이버는 호출 URI의 마지막 슬래시(/) 유무를 엄격히 구분하며, 이전의 틀린 설정을 가진 프로세스가 8000번 포트를 점유하여 새 설정을 차단했습니다.

### 3. 서버 인코딩 충돌
main.py가 윈도우 UTF-16 BOM 인코딩으로 저장되어, 첫 바이트(0xFF)를 인식 못한 파이썬 인터프리터가 서버 실행을 중단시켰습니다.

## 🧩 해결 및 보완 로직

### 100% 방어 코드
```javascript
if (!window.naver || !window.naver.maps) {
    console.warn("Naver Maps SDK Not Available - Sliding to Leaflet");
    this.fallbackToLeaflet(lat, lng); 
    return;
}
```

### 환경 정규화
- Port 8000 고정: 유령 프로세스 제거 후 8000번 단일 포트 사용
- UTF-8 No-BOM: 모든 파일을 표준 UTF-8로 재작성하여 안정성 확보

## ✅ 최종 상태
- 네이버 지도: 인증 실패 시 앱 중단 없이 Leaflet으로 자동 전환
- 상담 연동: JSON 파라미터 구조 불일치 및 서버 에러 해결
- URI 등록: NCP 콘솔에 모든 로컬 주소 조합 등록 완료
