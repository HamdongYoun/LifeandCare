# [aichat] 시스템 응급 상황 처리 규정

## 1. 응급 판단 로직 (Backend AI)
- 핵심 키워드: 심장, 가슴 통증, 호흡 곤란, 마비, 출혈, 의식 상실 등.
- 스코어링: 키워드 강도에 따라 emergencyAlert 타입 반환.

## 2. 앱 내 처리 우선순위
- UI 차단: 상담 중 모든 대화 입력을 차단하고 응급 경고를 최상단에 노출.
- 카운트다운(T-Minus 3s): 사용자가 의식이 없을 경우를 대비하여 자동 호출 대기.
- 위치 전송: (향후) 호출 시 사용자의 현재 좌표 정보를 함께 전달하는 기능 확장 고려.
