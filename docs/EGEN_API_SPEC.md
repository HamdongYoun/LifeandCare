# E-Gen API Module Specification

이 문서는 국립중앙의료원(응급의료포털 E-Gen) 연동 모듈의 기술 규격을 설명합니다.

## 1. 개요
본 모듈은 심사평가원(HIRA)의 병원 위치 정보와 국립중앙의료원의 실시간 가용 병상 정보를 결합하여 사용자에게 최신 응급실 상태를 제공합니다.

## 2. API 상세 정보
- **서비스명**: 국립중앙의료원_전국 응급의료기관 정보 조회 서비스
- **기능**: 실시간 응급실 가용병상 정보 조회 (`getEmrrmRltmUsefulSckbdInfoInqire`)
- **엔드포인트**: `http://apis.data.go.kr/B552657/HsptlAsembySearchService/getEmrrmRltmUsefulSckbdInfoInqire`
- **인증 방식**: API Key (공공데이터포털 발급 `EGEN_API_KEY`)

## 3. 데이터 로직 (Backend)
1.  **HIRA 조회**: 사용자의 위도/경도 기반 5km 반경 내 병원 TOP 5를 먼저 추출합니다.
2.  **E-Gen 호출**: 현재 시점의 전국(또는 주요 지역) 실시간 가용 병상 목록을 가져옵니다.
3.  **이름 기반 매칭(Matching Logic)**:
    - HIRA의 `yadmNm`(기관명)과 E-Gen의 `dutyName`(기관명)을 비교합니다.
    - `A in B` 또는 `B in A` 방식을 사용하여 '서울대학교병원'과 '서울대학교병원 (본원)' 같은 부분 일치 케이스를 대응합니다.
4.  **응답 데이터**:
    - `er`: 가용 병상 수 (예: "15석") 또는 "정보없음".
    - `status`: 현재 운영 상태 (기본: "진료중").

## 4. UI 렌더링 (Frontend)
- **상태 표시**: 가용 병상이 1개 이상인 경우 초록색 표시등(🟢), 없는 경우 빨간색(🔴)으로 시각화합니다.
- **실시간 갱신**: 사용자가 병원 검색을 수행할 때마다 백엔드를 통해 실시간 데이터를 다시 불러옵니다.

## 5. 관리 및 모니터링
- **타임아웃**: 실시간 정보의 특성상 응답 지연 시 사용자 경험을 해치지 않도록 5초의 타임아웃을 설정합니다.
- **환경 변수**: `server/.env` 파일의 `EGEN_API_KEY`를 통해 관리됩니다.
